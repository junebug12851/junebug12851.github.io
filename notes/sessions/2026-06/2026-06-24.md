## Design direction (v2) + implementation plan + warm mockup + day-centric blog

A long design/planning session with Fairy Fox. **No changes to the rendered site** —
everything landed in `notes/` (excluded from the build), shipped as `0.3.3`.

- **`notes/context/design-direction.md` (v2).** Studied joshwcomeau.com (homepage + his
  "How I Built My Blog"). Thesis: the site is the **legible surface of a decoupled,
  non-recursive mesh**. Settled: **decoupling as a UX law** (every page self-standing,
  flat, shallow, never lost; links are optional doors, never rails; no literal graph-map);
  unity via **consistent theming + shared component kit + wiki-style links**; **neutral =
  no ego, not low passion** (curatorial voice); a UI-copy rule (the page *is* its
  qualities, doesn't narrate them); meaningful motion + light interactivity welcome.
- **Keystone architecture confirmed (Option A):** site = a display reading ready-made
  `_data`; a **separate decoupled layer** (assistant + tooling + Actions) gathers across
  projects and writes it; one-directional. Static Jekyll now, **Laravel later** — only the
  display layer swaps.
- **`notes/plans/implementation.md`** — full build plan on the current 0.3.x codebase:
  two-layer `_data` contract, Sass-partial component kit, flat-web IA, Web-Component live
  examples (built *with* Fairy Fox), pulse, code presentation, phased roadmap. Added a
  **standardized status taxonomy**: lifecycle (alpha=red / beta=yellow / released=green) +
  version + activity (active/inactive), with correct current values.
- **`notes/mockups/v1-warm.html` (Phase 0.5).** Warm/friendly, Josh-grade-but-neutral
  language (Fraunces + Inter + JetBrains Mono; coral/honey; light + dark). Home portal
  with a 30-day commit chart + scrolling activity ticker, standardized node cards, and a
  full project-node page. Iterated heavily; dropped the DPL live example for now (needs
  Fairy Fox's input).
- **Day-centric blog model.** `notes/reference/blogging-workflow.md` rewritten: a post is
  a **day** (all projects touched + the talking/decisions; session notes fold in; a
  talk-only day still posts), and the **hub itself is a project** in those posts. Fixed an
  old "first person is fine" contradiction. Existing `_posts/` to be redone day-centric
  (preserve content) — see `plans/next-steps.md`.
- Threaded corrections through `principles.md`, `project.md`, `plans/future.md`,
  `next-steps.md`, and the `notes/README.md` map. Memory updated to thin pointers (project
  knowledge lives in these open notes).
