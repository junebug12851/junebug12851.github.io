### 2026-06-23 — Site redesign: full-bleed layout, refined AA theme, expanded docs (0.3.0)

A visual and structural overhaul of the public site, plus a much fuller
documentation library that surfaces this project's whole notes tree.

- **Layout & theme.** Rebuilt `assets/css/main.css` around a full-width fluid
  container and full-bleed section bands (alternating tints) instead of one narrow
  column; added a sticky footer (flex-column `body`, `main{flex:1}`,
  `footer{margin-top:auto}`) so short pages still anchor it to the bottom. Refined the
  violet/amber palette and tuned every text/background pair to meet **WCAG 2.1 AA** —
  verified, all ≥ 4.5:1 normal / ≥ 3:1 large (added `--accent-ink` for AA-safe violet
  text and an AA-safe primary gradient). Added `:focus-visible` rings and a
  reduced-motion guard.
- **Home page.** Stronger hero (eyebrow + headline + lede), uncramped 3-up project
  cards, a proper "Documentation library" band of category cards replacing the old
  three boxes, and a "Latest updates" card grid.
- **Navigation.** The Projects menu is now a plain list of project names linking to
  each project's on-site `/docs/<key>/` page, with an "All projects" footer. Renamed
  **Blog → Updates** across the nav, footer, post layout, and index.
- **Documentation expanded.** Surfaced the whole `notes/` tree into the library,
  separated from the shared/template standards: a new **"This site"** category
  (architecture, system map, deployment, updates & round-ups, design decisions,
  principles & voice) plus a **Documentation** standard, all rewritten in the neutral
  voice. Renamed the standards group to "Shared standards"; fixed project-doc ordering.
- **About** rewritten into a comprehensive, structured page; the `page` layout now uses
  a header band + body so pages aren't a narrow column.
