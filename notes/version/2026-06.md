### 2026-06-22 — Add a Downloads page with per-artifact tabs (0.2.2)

New `/downloads/` page giving each project a one-stop place to grab the latest
build artifacts, with condensed build/install/run instructions. For each project
there's a tab per build artifact (Windows installer/portable, Linux AppImage/tarball,
macOS, etc.) plus a **Source & build** tab; each tab pairs a direct download button
with the short version of how to install and run it. Every project header also links
out to its GitHub **release page** (the always-current fallback) and **README**.

- **Data:** new `_data/downloads.yml` (companion to `projects.yml`, joined by `key`)
  holds the download specifics only — direct file URLs are version-pinned to the
  current release and must be refreshed per release; the release-page link never
  goes stale. Covers `pokered-save-editor-2` (`v0.13.1-alpha`), the predecessor
  `pokered-save-editor` (`v2.0.1`, final), and `random-ai-prompt` (source/run only —
  it has no published binaries).
- **Page + UI:** `downloads.md` renders JS-free, accessible radio-button tabs; new
  `.dl-*` styles in `assets/css/main.css` reuse the existing design tokens.
- **Nav:** added **Downloads** to the header nav and a Downloads link to each card on
  `/projects/`. PATCH bump `0.2.1 → 0.2.2`.

### 2026-06-22 — Initial project: Jekyll home page + cross-project hub

Created `fairyfox.io` from scratch: a clean, custom (no external theme) Jekyll
site that serves as a personal home page and a hub that ties my repositories
together.

- **Site:** home, projects, blog, and about pages; a first blog post; custom
  responsive CSS (light/dark); RSS feed, SEO tags, and a sitemap via plugins.
- **Deploy:** GitHub Actions builds the site with Bundler + Jekyll and publishes
  to GitHub Pages on every push to `main` (no Pages-gem plugin limits). `CNAME`
  set for the `fairyfox.io` custom domain.
- **Hub:** `hub/` holds the cross-project standards, copy-paste templates, and
  the project registry that other repos pull on demand.
- **Notes:** adopted the living-notes system (status / sessions / changelog /
  context / systems / reference / decisions / plans) used across my projects.
- **Sync model:** documented the pull-only, git-only cross-project model and
  seeded `assets/references/` (git-ignored) with shallow clones of
  `pokered-save-editor-2` and `random-ai-prompt`.

Starting version: `0.1.0`.
