### 2026-06-25 — Docs-site standard: domain & publishing (0.5.1)

Added `hub/standards/docs-site/10-domain-and-publishing.md` — the infrastructure under
the one-domain model: how a project joins the `fairyfox.io` domain on GitHub Pages.
Key fact spelled out: because `fairyfox.io` is the custom domain on the **user site**
(`junebug12851.github.io`), every other repo's Pages site is **auto-served at
`fairyfox.io/<repo>/`** — a project joins just by enabling Pages, no per-project DNS,
**no CNAME in the project repo**. Per-project steps (enable Pages, set base path to
`/<repo>` for the generator, verify HTTPS, set the registry `docs:` URL) and the
gotchas (no project CNAME; base path must equal the slug or the theme's CSS/links
break; HTTPS; themed 404). Wired into the docs-site README index and the one-domain
section of `05-navigation-and-cross-linking.md`. PATCH `0.5.0 → 0.5.1`.

### 2026-06-25 — Docs-site design-system standard (0.5.0)

New shared standard `hub/standards/docs-site/` — a 10-file spec that lets every
project build and maintain a documentation site that's **seamless with fairyfox.io**:
same theme/colours/type/components, same structure, and required two-way linking
(project header/footer/breadcrumb back to Fairy Fox; the main site links in), all on
the one domain (`fairyfox.io/<key>/`). It's a **spec, not a theme package** —
AI-driven projects read it and reproduce the design in their own stack (per FF's
direction), matching as closely as the stack allows; generated API docs are
boundaried, not faked. Files: README + 01 overview/principles · 02 design tokens
(exact dark+light palette, Fraunces/Inter/JetBrains, spacing/shadow/motion) · 03
layout/structure · 04 components · 05 navigation/cross-linking · 06
content/organization · 07 accessibility (WCAG 2.1 AA) · 08 compliance checklist · 09
adopting/maintaining. **fairyfox.io is the master copy: exempt from auto-applied
style/theme updates — manual review by FF only, one-way outward to projects.** Wired
into `hub/README.md` and a new `CLAUDE.md` maintenance trigger. MINOR `0.4.12 → 0.5.0`.

### 2026-06-25 — Update command: gate on a named fairyfox reference (0.4.10)

Tightened how the check-for-updates flow is recognized. The gate is **a reference
to the fairyfox ecosystem** — "fairyfox" / "fairyfox.io" / "the fairyfox system",
"the hub", "the mesh", "the (shared) standards", or a named standard/runbook —

template + this repo's `CLAUDE.md`. **Naming:** the AI/command layer calls the mesh
**"the fairyfox system"**; the public **website keeps "hub"** (shorter); structural
paths (`hub/` folder, registry) keep "hub". Recognition stays natural-language, not
magic strings.
