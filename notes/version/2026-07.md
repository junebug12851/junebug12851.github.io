### 2026-07-06 — Fix: the `/games/` redirect stub flashed white en route to Games (0.14.5)

The "Games" nav link points at `/games/` — a redirect stub (`games.html`, `permalink: /games/`)
that meta-refreshes to `/fairyfox-games/`. The stub carried **no styling**, so the browser painted
it plain **white** for the instant before the refresh fired: a visible white flash every time you
opened Games from anywhere on the site (the rest of the site and the destination are dark, so it
stood out). Gave the stub the site's own early-paint treatment — a tiny inline script sets
`<html>`'s background/colour to the **reader-resolved theme** (`#181017` dark / `#efe4d1` light /
`#f1e3c2` sepia, via the shared origin-wide `fairyfox:reader:b` key) before the refresh, with a
static dark fallback + `theme-color` metas. The redirect itself is unchanged; the stub now matches
the theme instead of flashing white. Verified with a Jekyll build + Chrome (computed `<html>`
background `rgb(24,16,23)`, no white frame). VERSION → 0.14.5 (PATCH).

(Context: reported against Fairy Fox Games, but the flash was never in that repo — the stub lives
here in the hub. The fairyfox-games pages got defensive early-background hardening too, v0.18.1/2.)

### 2026-07-05 — Reader redesign: contrast pass, accent picker, real text scaling (0.14.0)

A MINOR milestone reworking the reader + theming after the first cut read flat and the
size/spacing controls didn't visibly do anything.

- **Contrast pass on all three themes.** Dark, light and sepia were retuned for real
  surface separation (panels raised a clear step off the ground, stronger hairlines) and
  brighter/darker secondary text, so nothing looks washed-out. **Sepia** in particular was
  rebuilt into a proper warm-paper look with near-ink brown text. Tuned to WCAG 2.1 AA.
- **Text size now scales the whole page.** The reader's size lever sets the document
  **root font-size** (`html`), so the entire rem-based UI resizes — it visibly works on
  every page, not just prose. The confusing `A−`/`A+` pair is replaced by a **5-step
  segmented "A" scale** that shows the current level. Line spacing drives `body`
  line-height; width still caps the reading measure.
- **Accent colour picker.** A row of colour circles (plus a "Default" that reverts to the
  theme accent) lets the visitor recolour the site accent. A picked hex overrides
  `--accent`/`--violet` and derives readable `--accent-ink`/`--link` by mixing toward
  `--text`, so it stays legible in every theme.
- **A little more colour** where it helps: an accent bar on section headings, an accent
  italic hero ampersand, and a fixed stray hard-coded button-hover purple.
- **New storage key `fairyfox:reader:b`.** The schema changed (accent + root-scale size),
  so the key is versioned with a `:b` suffix — it won't collide with the older
  `fairyfox:reader` a sibling may still use until it adopts this model.
- **Standard synced** (`hub/standards/docs-site/` 02/04/11 + `reference/main.css`): the new
  key/model, the accent picker, the 5-step size scale, and the retuned palettes.
