### 2026-07-02 — Standing "adopt standards by default" authorization (0.12.1)

Made node adoption of hub standards the **default** posture, on the owner's instruction
("getting the nodes to adopt the standards and policies is very important — it should be
default"). Added a **standing express-authorization** (`adopt-standards-by-default`) to
[`hub/authorizations.yml`](../../hub/authorizations.yml) covering **all** of
`hub/standards/` and `hub/templates/`: when a node runs the fairyfox check/adopt flow,
any pending standard/template change is pre-authorized — it **adopts by default** (report
*and* apply) instead of reporting-and-waiting. Made this explicit in the `CLAUDE.md`
template and the `adopting-updates` runbook.

**Anti-recursion is untouched** (owner chose the "standing pre-authorization" option, not
scheduled checks): adoption still happens **only when the node runs the flow** — no
auto-pull, no scheduled cross-repo sync, no hub→node push — and the grant removes **only**
the redundant confirmation pause. Every other safety step still runs in full
(reconcile-not-clobber, divergence re-prompt, process report, reviewable commit, full
verification before and after; a node still bootstraps its first express-auth adoption
under the normal pause).

### 2026-07-02 — Fold RAP's proposals into six new/updated standards (0.12.0, MINOR)

Report-review fold-back: Random AI Prompt's six undigested `notes/fairyfox-reports/`
proposals adopted mesh-wide as mandatory standards (owner go-ahead in a live session;
two scope calls decided by the owner — **branch protection mandatory for all repos**,
**legal-docs mandatory for all repos**). The automated inbound briefing
(`INBOX-2026-07-02-0202.md`) had synthesized the same themes.

**git-workflow (fixed a defect):** the release flow never brought `main` back to `dev`,
so `dev` drifted a commit behind every release (RAP hit **32 behind**). Added the
invariant *"after any release, `dev` must contain `main`"* — PATCH/MINOR/MAJOR
fast-forward `dev` to `main`, hotfix does a real merge; reinforced never-commit-on-`main`;
noted feature branches aren't CI-tested (the `dev` merge is the gate); added a **PR-based
release path** for branch-protected repos. New `templates/branch-sync.yml` guard fails when
`dev` drifts behind `main`. Verify rows added.

**Five new standards** (each with a `## Verify` slice + a `compliance.md` matrix row):
- `supply-chain-hardening.md` — least-privilege workflow permissions, SHA-pinned Actions,
  `SECURITY.md`, signed releases, **mandatory branch protection** (solo config), stated
  solo ceiling (~8/10) + badge lag. Reconciled with git-workflow's PR release.
- `dependencies.md` — upgrade aggressively behind a real local test gate; Dependabot →
  `dev`; fix-don't-pin; bump on artifact change.
- `legal-docs.md` — self-hosted, code-accurate Privacy/Terms/Cookies, kept current;
  mandatory for **all** repos.
- `agent-tooling.md` — PowerShell + file tools, never the bash sandbox; execute don't
  hand off; `.gitattributes` line-ending hygiene.
- `badges.md` — the canonical grouped shields.io set, as many as apply per project.

**process-reports.md:** named the canonical **node → hub proposal** report shape
(`procedure: propose-standard`, required front matter) — the five RAP proposals arrived
with inconsistent/missing front matter.

**New templates:** `project.gitattributes`, `SECURITY.md`, `dependabot.yml`,
`branch-sync.yml`, `README-badges.md`, `legal/{privacy,terms,cookies}.html`.
**Establishment:** wired into `compliance.md`, the `CLAUDE.md` template (tooling rule +
back-merge release + legal notes-trigger), and all three lifecycle runbooks (setup /
onboard / adopt copy the new templates + enable branch protection). `hub/README.md` +
`templates/README.md` updated. RAP's `reports_through` in `.last-seen.yml` advanced past
the six digested reports. Jekyll build green. Out of scope: the fairyfox-games
